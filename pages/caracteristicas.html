O Android é um sistema operacional multitarefa [Girollete, 2012], o que permite ao usuário trabalhar com vários aplicativos ao mesmo tempo sem nenhuma dificuldade. [Machado and Maia, 2007] Ele também é multiusuário [Andrade, 2011], ou seja, múltiplos usuários podem utilizar o sistema simultaneamente.[Machado and Maia, 2007]

Com o lançamento do API de desenvolvimento(que será apresentada adiante), os programadores passaram a ter facilidade no acesso a especificações do Android, podendo alterar o código conforme suas necessidades, ou até mesmo estudá-lo para construir aplicações mais eficientes. [Andrade, 2011]

Apesar da plataforma utilizar os layouts tradicionais de smartphone, é facilmente adaptável a dispositivos VGA maiores(layouts responsivos). O armazenamento de informações é feito com uma biblioteca muito eficiente, a SQLite, que foi desenvolvida em linguagem C, e vem com um banco de dados embutido. O Android também é capaz de enviar mensagem de texto (SMS) ou também mensagens de texto com multimídia encorporada (MMS). O navegador do Android é baseado no WebKit que é um interpretador feito em C++ pela Apple e utilizado na maioria dos navegadores para Smartphones, Desktops, Tablets, Notebooks e Netbooks. As aplicações escritas em Android são escritas em Java (e são multiplataforma), e o Android possui uma máquina virtual eficiente, a Dalvik que suporta qualquer aplicação escrita em Java, ou seja, softwares desktop escrito em Java, são facilmente adaptados a essa plataforma. O sistema suporta a grande maioria dos formatos de áudio e vídeo como por exemplo mp3 e avi. O Android é totalmente capaz de utilizar dispositivos de hardware adicionais como telas touch-screen, GPS, câmeras de vídeo, etc. [Soares, 2010]

Antes de apresentar a API de desenvolvimento do Android(SDK), é necessário explicar alguns conceitos sobre API de desenvolvimento. A sigla API significa Apliccation Programming Interface ou Interface de Programação de Aplicativos e é um conjunto de normas e rotinas estabelecidos por um software para os desenvolvedores que desejam criar outras aplicações utilizando os serviços deste software. A API de desenvolvimento do Android (SDK) inclui um emulador, ferramentas de debug e existe um plugin para o Eclipse(uma das mais famosas IDE's para desenvolvimento Java), deixando tudo mais fácil para o desenvolvedor de aplicativos desktop, desenvolver para Android. [Soares, 2010]
